class VehicleAgent.BusAgent(vehicleAgent_obj, capacity, loading, start_station, end_station, bus_complete_sp, bus_complete_sp_dist, bus_stop_route, bus_stop_route_schedule, bus_route_naming)[source]

Bases: VehicleAgent

Parameters:

  • vehicleAgent_obj – the vehicle object reference

  • capacity – integer, the maximum capacity

  • loading – integer, the bus’s current loaded number of people (including driver)

  • start_station – the bus’s start station (integer, shp_G node id)

  • end_station – the bus’s end station (integer, shp_G node id)

  • bus_complete_sp – the bus’s current shortest path routing queue (list of integers representing shp_G node ids)

  • bus_complete_sp_dist – the bus’s current shortest path break-down link length (list of floats representing each link’s length)

  • bus_stop_route – list of integer shp_G node ids, not necessarily to be continuous

  • bus_stop_route_schedule – the strict one-to-one mapping for bus_stop_route list. example: [0, 360, 720, 1000] means the bus should arrive node 49 at tick 0

assign_attributes(agent_id, node_id_mutex, edge_id_mutex, long, lat, speed, destination, status, routing_queue, routing_queue_dist, routing_queue_cross_process_node_id, belonging_rank, capacity, loading)[source]

Due to code design, a vehicle agent’s attributes will be assigned HERE, using this function

Parameters:

  • agent_id – the vehicle’s universe ID (rather than process’s local ID)

  • node_id_mutex – the vehicle’s spawn shp_G node id (int)

  • edge_id_mutex – (this value is actually not used)

  • long – the vehicle’s spawn longitude (UTM, float)

  • lat – the vehicle’s spawn latitude (UTM, float)

  • speed – the vehicle’s speed (float)

  • destination – the vehicle’s destination (shp_G node id, integer)

  • status – the vehicle’s status (integer), currently we use 0 as “idle”, 1 as “more”, 2 as “waiting at the boundary to be transferred)

  • routing_queue – the vehicle’s current shortest path routing queue (list of integers representing shp_G node ids)

  • routing_queue_dist – the vehicle’s current shortest path break-down link length (list of floats representing each link’s length)

  • routing_queue_cross_process_node_id – updated at: 1. init 2. re-partitioning, to decide which node is the decisive boundary node to wait and be transferred

  • belonging_rank – integer, the vehicle’s belonging rank
